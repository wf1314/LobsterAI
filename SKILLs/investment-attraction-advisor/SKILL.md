# 招商企业推荐 (Investment Attraction Advisor)

## 概述

面向招商引资场景，支持三类任务：

1. **主动推荐**：根据园区/地区招商条件，检索并筛选目标企业。
2. **落地评估**：评估指定企业与招商目标的匹配度和落地价值。
3. **批量对比**：对意向企业清单进行评分、排序和策略建议。

最终输出应以企业数据为依据，给出企业推荐、评分排序、落地价值判断和招商行动建议。

## 工作流程

### Step 0：信息收集（必须）

开始企业查询前，判断用户是否已提供足够的招商背景。已提供的信息直接使用；只追问会影响查询或评分的关键缺口。

关键缺口包括：

- 招商方：园区/地区、所在城市、产业方向、核心政策、目标企业规模或资质要求。
- 企业评估：被评估企业全称、评估重点。
- 批量对比：企业名单、排序目标、是否需要完整报告。

---

### Step 1：企业数据查询

查询前按“资源加载规则”读取所需 reference 文件，然后使用 MCP 企业数据工具查询。根据任务模式选择查询策略：

**模式 A：主动推荐模式**
- 先用 `industry_mapping.md` 将产业方向转为查询关键词、细分赛道和资质标签。
- 每批查询 5-10 家候选企业，可按结果质量继续迭代。
- 优先保留经营状态正常、规模符合要求、主业与招商方向匹配的企业。

**模式 B：企业评估模式**
- 精准查询指定企业的工商、股权、经营、资质、知识产权、对外投资等信息。
- 围绕用户关注点补充与落地可行性相关的数据。

**查询注意事项：**
- 优先使用已连接的 MCP 工具
- 若 MCP 工具未连接，使用 WebSearch 联网搜索补充，并告知用户数据来源
- 记录每家企业的原始查询结果，避免重复查询

---

### Step 2：企业初筛与评分

评分前必须读取 `references/scoring_guide.md`，并按其中的维度、权重公式、打分标准和缺失数据处理规则执行。

评分要求：

- 每家企业给出各维度分数、简短理由和综合得分。
- 用户对某维度有特殊要求时，优先遵循用户标准，并说明对评分的影响。
- 数据不足时标注“数据不足”或“待核实”，不要编造缺失指标。
- 推荐优先级应由综合得分、招商价值和落地可行性共同决定。

---

### Step 3：生成推荐报告

生成完整报告前必须读取 `references/report_template.md`，并以该模板为最终结构依据：

- 严格保留模板的结构，不要擅自删减。
- 将模板中的 `[占位符]` 替换为实际内容，不保留占位符说明。
- 表格字段以模板为准；若数据缺失，写“待核实”或“数据不足”，不得编造。
- 报告编制日期使用当前日期，如有时间日期工具优先调用工具获取当前日期。
- 对企业事实、评分判断、预估数据分别标注来源或依据。

---

### Step 4：输出格式规范
。
- 语气专业、客观、克制，避免过度营销化表达。
- 对 MCP 查询、公开信息、推断判断、预估数据做清晰区分。
- 不编造企业数据、联系方式、扩张计划、税收贡献或政策细节。

## 资源文件速查

- `references/report_template.md`：推荐报告完整模板，含占位符说明。生成完整报告前必须读取。
- `references/industry_mapping.md`：常见招商产业方向与对应行业分类映射表。主动推荐或产业关键词不明确时必须读取。
- `references/scoring_guide.md`：企业评分详细标准与打分示例。评分、排序或优先级判断前必须读取。

## 任务路由示例

- “推荐 5-8 家目标企业” → 读取 `industry_mapping.md`、`scoring_guide.md`；若需要报告，再读取 `report_template.md`。
- “评估某企业是否适合落地” → 查询指定企业，读取 `scoring_guide.md`；若要正式报告，再读取 `report_template.md`。
- “评估这批企业并排序” → 批量查询，读取 `scoring_guide.md`；输出对比表或按需生成报告。
